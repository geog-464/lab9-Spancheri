Q2: Why is the JS script loaded at the end of the html file and not the beginning like the stylesheet?
HTML loads from top to bottom (single-thread language, runs one command at a time) - the head will load first, followed by the body and then everything inside the body. If were to put the JS script in the head, the entire JS will load before the HTML. This would reduce the speed of loading of your page ( due to having to run all of JS before any of the HTML elements) as well as cause potential erros (there may be JS code that alters the HTML - but those HTML element won't yet be availble to alterate, due to being run after).

Q3: What is the onload method doing? Why is it good to load JavaScript in this way?
The onload method occurs when an object has been loaded. This method is usually used within the body element to execute a script after a web page has loaded all its content (such as images, scripts, CSS files, etc.)
